## 4. 모델별 결과 요약

### GLM-5.2

장점:

- 기본 회의록 요약에서 지정 형식과 Action Items 표를 안정적으로 지켰다.
- 기한이 없는 항목을 `확인 필요`로 처리했고, 출장비 규정 질문에도 확답하지 않았다.
- 외부 공유용 변환에서 API 응답 시간과 QA 상세 수치를 제거했다.
- 계산 테스트에서 QA 완료율 약 73.8%, 잔여 11개, API 초과 1.8초를 정확히 계산했다.

한계:

- 모호한 입력에서 `법무는 아직`을 `법무 검토는 아직 진행되지 않음`으로 다소 강하게 해석했다.
- 조건 변경 테스트에서 `대시보드 필터`를 `대시보드 필더`로 쓰는 오타가 있었다.
- 외부 공유용 문안이 다소 짧아, 협력사 요청사항까지 풍부하게 정리되지는 않았다.

판정:

- 현 테스트에서는 GLM-5.2가 더 안정적인 선택이다.
- 특히 환각 억제와 외부 공유용 민감 정보 제거에서 DeepSeek-V4-Pro보다 안전했다.

### DeepSeek-V4-Pro

장점:

- 기본 요약의 문장 품질과 구조화 능력이 좋았다.
- 모호한 입력 대응에서 확인 질문 3개와 초안을 함께 제공했다.
- 계산 테스트와 출장비 규정 환각 방지 테스트는 정확히 통과했다.

한계:

- 기본 회의록 요약에서 `이번 주 금요일 전`을 `7월 11일(금) 전`으로 잘못 변환했다. 2026년 7월 7일 기준 금요일은 7월 10일이므로 날짜 추론 오류다.
- 외부 공유용 민감 정보 제거 테스트에서 `현재 70% 진행`이라는 내부 수치를 남겼다.
- 조건 변경 테스트에서 임원 보고용으로 바꾸라는 요청에도 표를 유지해, 세부 표 축소 요구를 완전히 반영하지 못했다.

판정:

- 답변 품질은 높지만, 날짜 추론과 민감 정보 제거에서 실무 리스크가 있었다.
- 외부 공유나 사실 검증이 중요한 회의록 자동화 과업에서는 추가 프롬프트 보강이 필요하다.

## 5. 최종 결론

현 단계의 Ollama API 2종 비교에서는 GLM-5.2를 우선 선택한다.

1. GLM-5.2는 불확실한 기한과 정책 질문을 `확인 필요`로 처리해 환각 억제 점수가 높았다.
2. 외부 협력사용 변환에서 내부 성능 수치와 QA 상세 수치를 제거해 공개 범위 제어가 더 안정적이었다.
3. DeepSeek-V4-Pro는 문장 품질은 좋았지만, 날짜를 잘못 추론하고 민감 수치를 남기는 문제가 있어 실무 적용 전 추가 검증이 필요하다.

이 파일은 Ollama API 기반 2종 비교 결과만 다룬다. 3종 최종 비교 결과와 최종 선정 결론은 `results/model_comparison_summary_20260721.md`와 `docs/04_final_report.md`에 정리했다.
